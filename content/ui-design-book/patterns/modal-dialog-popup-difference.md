# Modal / Dialog / Popup の違い

## 1. 目的 / どんな体験を支えるか

Modal、Dialog、Popupは、いずれもユーザーの注意を引き、特定の情報や操作を提示するUIパターンです。それぞれの目的は異なります：

- **Modal**: ユーザーの注意を強制的に引き、重要な操作や情報を提示する
- **Dialog**: ユーザーとの対話（確認、選択、入力）を促す
- **Popup**: 補足情報や追加コンテンツを一時的に表示する

適切に使い分けることで、ユーザー体験を向上させ、操作の混乱を防ぎます。

## 2. 構造

### Modal

```
[Backdrop (半透明の背景)]
  └─ [Modal Container]
      ├─ [Header]
      │   ├─ [Title]
      │   └─ [Close Button]
      ├─ [Content]
      └─ [Footer]
          ├─ [Cancel Button]
          └─ [Action Button]
```

### Dialog

```
[Backdrop]
  └─ [Dialog Container]
      ├─ [Message / Question]
      └─ [Action Buttons]
          ├─ [Cancel]
          └─ [Confirm]
```

### Popup

```
[Trigger Element]
  └─ [Popup Container] (相対位置)
      └─ [Content]
```

## 3. デザインの鉄則

### Modal

- **背景のブロック**: 背景を暗くして、モーダルに注意を向ける
- **中央配置**: 画面中央に配置し、視覚的に目立たせる
- **閉じる方法**: 複数の方法を提供（Xボタン、背景クリック、ESCキー）
- **サイズ**: コンテンツに応じた適切なサイズ（最大幅を設定）

### Dialog

- **簡潔性**: メッセージは簡潔に、質問は明確に
- **アクションの明確化**: ボタンのラベルで操作の結果が分かるように
- **デフォルト選択**: 安全な選択肢をデフォルトにする（削除確認など）

### Popup

- **非侵入的**: ユーザーの操作を妨げない
- **適切な位置**: トリガー要素の近くに表示
- **自動非表示**: 一定時間後や外部クリックで閉じる

## 4. よくある失敗と対策

### 失敗例1: Modalの過度な使用

**問題**: すべての情報をModalで表示する

**対策**: 
- 重要度の低い情報はインライン表示やツールチップを使用
- Modalは本当に必要な場合のみ使用

### 失敗例2: 閉じる方法が不明確

**問題**: 閉じるボタンが見つからない、または1つの方法しかない

**対策**:
- 複数の閉じる方法を提供（Xボタン、背景クリック、ESCキー）
- 閉じるボタンを明確に表示

### 失敗例3: モバイルでの表示問題

**問題**: デスクトップ用のサイズで、モバイルでは画面からはみ出す

**対策**:
- レスポンシブデザインで、モバイルでは全幅表示
- スクロール可能にする

## 5. 事例（一般的なUI）

### Modalの使用例

- **画像の拡大表示**: ギャラリーで画像をクリックした際の拡大表示
- **フォーム入力**: 詳細な設定や編集フォーム
- **重要な通知**: システムメンテナンスのお知らせ

### Dialogの使用例

- **削除確認**: 「本当に削除しますか？」の確認
- **保存確認**: 変更を保存するかどうかの確認
- **選択ダイアログ**: 複数の選択肢から選ぶ

### Popupの使用例

- **ツールチップ**: アイコンの説明
- **ドロップダウンメニュー**: アクションメニュー
- **通知バナー**: 一時的な成功/エラーメッセージ

## 6. 関連するコンポーネント

- **Button**: モーダル内のアクションボタン
- **Input Field**: モーダル内のフォーム入力
- **Backdrop**: モーダルの背景オーバーレイ
- **Icon**: 閉じるボタンのアイコン
