# Button

## 1. 何のためのコンポーネントか（役割）

Buttonは、ユーザーが特定のアクションを実行するためのインタラクティブなUI要素です。主な役割は以下の通りです：

- **アクションの実行**: フォーム送信、データ保存、削除、承認などの操作を開始する
- **ナビゲーション**: 次のステップやページへの遷移を促す
- **状態の変更**: 設定の有効化/無効化、表示/非表示の切り替えなど

Buttonは、ユーザーが「何かを行う」ための明確な導線を提供します。

## 2. よく使う場面

- **フォーム送信**: ログイン、登録、問い合わせフォームなど
- **確認ダイアログ**: 削除、保存、公開などの重要な操作の確認
- **ナビゲーション**: 「次へ」「戻る」「完了」などのステップ進行
- **アクションバー**: ツールバーやフッターに配置される主要操作
- **カード内アクション**: 商品の購入、記事の保存、シェアなど

## 3. 構造と状態

### 基本構造

```
[Button Label]
```

### 主要な状態

- **Default（通常）**: 初期状態。クリック可能
- **Hover（ホバー）**: マウスオーバー時の視覚的フィードバック
- **Active（アクティブ）**: クリック中の状態
- **Disabled（無効）**: 操作ができない状態。グレーアウト
- **Loading（読み込み中）**: 処理実行中の状態。スピナーやプログレス表示

### バリエーション

- **Primary**: 主要なアクション。目立つ色（通常はブランドカラー）
- **Secondary**: 補助的なアクション。控えめなスタイル
- **Tertiary**: 第三の選択肢。テキストリンクに近い見た目
- **Danger**: 破壊的な操作（削除など）。警告色を使用
- **Ghost**: 背景なし。最小限のスタイル

## 4. デザインのポイント

### 視覚的階層

- **サイズ**: 重要度に応じてサイズを変える（Large / Medium / Small）
- **色**: Primaryは目立つ色、Secondaryは控えめな色を使用
- **コントラスト**: 背景とのコントラスト比はWCAG AA基準（4.5:1以上）を満たす

### ラベル設計

- **動詞で始める**: 「保存する」「削除する」「送信する」など、動作を明確に
- **簡潔に**: 長すぎるラベルは避ける（通常2-4語程度）
- **一貫性**: 同じ操作には同じラベルを使用

### レイアウト

- **間隔**: 複数のボタンがある場合、適切な間隔を確保（通常8-16px）
- **配置**: 主要なアクションは右側または下部に配置（左から右への視線の流れ）

## 5. アクセシビリティ

### キーボード操作

- **Tabキー**: フォーカス可能にする
- **Enter/Space**: クリックと同じ動作を実行
- **フォーカス表示**: キーボードフォーカス時に明確な視覚的インジケーターを表示

### ARIA属性

- `role="button"`: ボタンであることを明示（必要に応じて）
- `aria-label`: アイコンのみのボタンには説明を追加
- `aria-disabled="true"`: 無効状態をスクリーンリーダーに伝える
- `aria-busy="true"`: 読み込み中であることを示す

### その他

- **十分なサイズ**: タッチターゲットは最低44×44px（モバイル）
- **色だけに依存しない**: 無効状態は色だけでなく、アイコンやテキストでも示す

## 6. 誤用・アンチパターン

### よくある間違い

1. **リンクとボタンの混同**
   - ページ遷移はリンク、アクション実行はボタンを使う
   - 誤: `<button onclick="location.href='/page'">ページへ</button>`
   - 正: `<a href="/page">ページへ</a>`

2. **過度な装飾**
   - アニメーションやエフェクトが多すぎると、操作の意図が不明確になる

3. **無効状態の説明不足**
   - なぜ無効なのか、どうすれば有効になるのかを明示しない

4. **複数のPrimaryボタン**
   - 主要なアクションは1つに絞る。複数あるとユーザーが迷う

5. **小さすぎるタッチターゲット**
   - モバイルで操作しづらくなる

## 7. 関連するUIパターン

- **Form**: フォーム内での送信ボタンとして使用
- **Modal / Dialog**: 確認ダイアログのアクションボタン
- **Toolbar**: ツールバー内の操作ボタン
- **Card**: カード内のアクションボタン
- **Navigation**: ナビゲーションの進行ボタン
