# Search Input

## 1. 何のためのコンポーネントか（役割）

Search Inputは、検索クエリを入力するための専用の入力フィールドです。主な役割は：

- **検索クエリの入力**: ユーザーが検索したいキーワードを入力
- **検索の開始**: 入力したクエリで検索を実行
- **検索候補の表示**: 入力中に検索候補を表示（オプション）

## 2. よく使う場面

- **サイト内検索**: サイト全体の検索機能
- **商品検索**: ECサイトでの商品検索
- **コンテンツ検索**: ブログやドキュメントの検索
- **ユーザー検索**: SNSでのユーザー検索

## 3. 構造と状態

### 基本構造

```
[Search Icon] [Input Field] [Clear Button] [Search Button]
```

### 主要な状態

- **Default**: 初期状態（プレースホルダー表示）
- **Typing**: 入力中
- **Filled**: 値が入力された状態
- **Loading**: 検索実行中
- **Results**: 検索結果表示中

## 4. デザインのポイント

- **検索アイコン**: 検索機能であることを視覚的に示す
- **クリアボタン**: 入力内容を簡単にクリアできる
- **検索ボタン**: 検索実行のためのボタン（オプション）
- **オートコンプリート**: 入力中に候補を表示

## 5. アクセシビリティ

- **ラベル**: `aria-label`で「検索」であることを示す
- **キーボード操作**: Enterキーで検索実行
- **フォーカス表示**: 明確な視覚的インジケーター

## 6. 誤用・アンチパターン

- **プレースホルダーのみ**: ラベルも提供する
- **検索ボタンがない**: モバイルでは検索ボタンがあると分かりやすい

## 7. 関連するUIパターン

- **Search**: 検索機能全体のパターン
- **Autocomplete**: 自動補完機能と組み合わせ
- **Filter**: フィルター機能と組み合わせ
