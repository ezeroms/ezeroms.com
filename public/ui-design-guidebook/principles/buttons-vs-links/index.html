    Buttons vs Links：役割の違い

    ボタンとリンクの役割の違いを理解し、適切に使い分けるための原則。

    Buttons vs Links：役割の違い

    1. 原則の定義

    **Button（ボタン）とLink（リンク）**は、見た目が似ていても、その役割と使用目的が根本的に異なります：

    • Button: アクションを実行する（データの送信、状態の変更、操作の開始）
    • Link: 場所に移動する（ページ遷移、アンカーリンク、外部リソースへのアクセス）

    この違いを理解し、適切に使い分けることで、ユーザーは操作の結果を予測でき、アクセシビリティも向上します。

    2. なぜ重要か（背景・認知科学）

    ユーザーの期待

    ユーザーは、UI要素の見た目からその動作を予測します：

    • リンクを見る: 「別のページに移動する」と期待
    • ボタンを見る: 「何かの操作を実行する」と期待

    この期待に反すると、ユーザーは混乱し、操作に迷いが生じます。

    アクセシビリティ

    スクリーンリーダーなどの支援技術は、要素の種類（<button> vs <a>）に基づいて動作を変えます：

    • リンク: 「リンク、[テキスト]」と読み上げ、リンク一覧に表示
    • ボタン: 「ボタン、[テキスト]」と読み上げ、フォームコントロールとして扱う

    適切な要素を使用することで、支援技術ユーザーも正しく操作できます。

    SEOとブラウザの動作

    • リンク: 検索エンジンがクロール可能、右クリックで「新しいタブで開く」などの操作が可能
    • ボタン: 検索エンジンはクロールしない、右クリックのコンテキストメニューが異なる

    3. 適用シーン

    Buttonを使うべき場合

    • フォームの送信
    • データの保存、削除、更新
    • モーダルの開閉
    • 設定の有効化/無効化
    • フィルターの適用
    • ソート順の変更

    Linkを使うべき場合

    • 別のページへの遷移
    • 外部サイトへのアクセス
    • 同一ページ内のアンカーリンク
    • ファイルのダウンロード
    • メールアドレスへのリンク（mailto:）

    境界線上のケース

    • 「詳細を見る」: ページ遷移なのでLink
    • 「商品をカートに追加」: アクション実行なのでButton
    • 「次へ」ボタン（ウィザード）: ページ遷移だが、ウィザードの進行なのでButton（ただし、URLが変わる場合はLinkも検討）

    4. 判断基準 / デザインチェック

    判断フローチャート

    1. URLが変わるか？

      • Yes → Linkを検討
      • No → Buttonを検討

    2. データが送信されるか？

      • Yes → Button
      • No → Linkを検討

    3. 状態が変更されるか？

      • Yes → Button
      • No → Linkを検討

    4. ページ遷移のみか？

      • Yes → Link
      • No → Buttonを検討

    デザインチェックリスト

    • フォーム送信にはButtonを使用しているか？
    • ページ遷移にはLinkを使用しているか？
    • 見た目だけでなく、適切なHTML要素を使用しているか？
    • スクリーンリーダーで正しく読み上げられるか？

    5. 誤用例

    誤用例1: リンクに見えるがボタンとして実装

    <!-- 誤 -->
<button onclick="location.href='/page'">ページへ</button>

<!-- 正 -->
<a href="/page">ページへ</a>

    誤用例2: ボタンに見えるがリンクとして実装

    <!-- 誤 -->
<a href="#" onclick="submitForm()">送信</a>

<!-- 正 -->
<button type="submit">送信</button>

    誤用例3: スタイルで見た目を変えすぎる

    リンクをボタンのように見せたり、ボタンをリンクのように見せたりすると、ユーザーの期待と実際の動作がずれます。

    6. 関連原則

    • Information Hierarchy（情報の階層構造）: ボタンとリンクの視覚的階層
    • 予測可能性（Predictability）: ユーザーが操作の結果を予測できること
    • アクセシビリティの基本: 適切なHTML要素の使用
    • 一貫性（Consistency）: 同じ操作には同じ要素を使用
