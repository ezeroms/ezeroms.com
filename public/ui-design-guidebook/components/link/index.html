    Link

    ページ間の遷移や外部リソースへのアクセスを提供するナビゲーション要素。

    Link

    1. 何のためのコンポーネントか（役割）

    Linkは、ユーザーを別のページやリソースに導くためのナビゲーション要素です。主な役割は：

    • ページ遷移: サイト内の別ページへの移動
    • 外部リソース: 外部サイトやドキュメントへのアクセス
    • アンカーリンク: 同一ページ内の特定セクションへのジャンプ
    • ダウンロード: ファイルのダウンロードを開始

    Linkは情報の関連性を示し、ユーザーの探索を支援します。

    2. よく使う場面

    • ナビゲーションメニュー: サイトの主要セクションへの導線
    • 本文中の参照: 記事内の関連ページへのリンク
    • フッターリンク: 利用規約、プライバシーポリシーなど
    • 関連記事: コンテンツページの関連リンク
    • 外部参照: 参考資料やソースへのリンク

    3. 構造と状態

    基本構造

    [Link Text]

    主要な状態

    • Default: 通常のリンク状態（通常は青色で下線）
    • Visited: 訪問済みリンク（通常は紫色）
    • Hover: マウスオーバー時の状態
    • Active: クリック中の状態
    • Focus: キーボードフォーカス時の状態

    4. デザインのポイント

    視覚的識別

    • 下線: テキストリンクには下線を使用（特に本文中）
    • 色: ブランドカラーや標準的なリンク色（通常は青系）
    • ホバー効果: マウスオーバー時に色を変える、下線を太くするなど

    ラベル設計

    • 説明的: 「ここをクリック」ではなく、リンク先の内容を示す
    • 簡潔: 長すぎるテキストは避ける
    • 文脈: 周囲のテキストからリンクの目的が理解できるように

    5. アクセシビリティ

    キーボード操作

    • Tabキー: フォーカス可能にする
    • Enter: リンクを開く
    • フォーカス表示: 明確な視覚的インジケーター

    ARIA属性

    • aria-label: リンクテキストだけでは不十分な場合に説明を追加
    • aria-current: 現在のページを示す（ナビゲーション内で）

    その他

    • 外部リンクの明示: 外部サイトへのリンクはアイコンやtarget="_blank"で示す
    • 十分なサイズ: タッチデバイスでも操作しやすいサイズを確保

    6. 誤用・アンチパターン

    よくある間違い

    1. ボタンとリンクの混同

      • アクション実行はボタン、ページ遷移はリンクを使う

    2. 「ここをクリック」

      • リンクテキスト自体が説明的であるべき

    3. 色だけに依存

      • 色覚異常者にも分かるよう、下線やアイコンも使用

    4. リンクの開き方

      • 外部リンクはtarget="_blank"とrel="noopener noreferrer"を使用

    7. 関連するUIパターン

    • Navigation: ナビゲーションメニュー内のリンク
    • Breadcrumbs: パンくずリストのリンク
    • Card: カード内のリンク
    • Footer: フッターのリンク群
