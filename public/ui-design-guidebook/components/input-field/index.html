    Input Field

    ユーザーがテキストや数値を入力するためのフォーム要素。

    Input Field

    1. 何のためのコンポーネントか（役割）

    Input Fieldは、ユーザーがテキストや数値などの情報を入力するためのフォーム要素です。主な役割は：

    • データ入力: 名前、メールアドレス、パスワードなどの情報を収集
    • 検索: 検索クエリの入力
    • フィルタリング: 条件の入力によるデータの絞り込み
    • 設定: アプリケーションの設定値の入力

    2. よく使う場面

    • ログインフォーム: ユーザー名、メールアドレス、パスワードの入力
    • 登録フォーム: 新規登録時の各種情報入力
    • 検索バー: サイト内検索や商品検索
    • 設定画面: アプリケーションの各種設定
    • 問い合わせフォーム: お問い合わせ内容の入力

    3. 構造と状態

    基本構造

    [Label]
[Input Field]
[Helper Text / Error Message]

    主要な状態

    • Default: 初期状態
    • Focus: フォーカス時（入力可能）
    • Filled: 値が入力された状態
    • Error: バリデーションエラー時
    • Disabled: 無効状態
    • Read-only: 読み取り専用

    バリエーション

    • Text: 通常のテキスト入力
    • Email: メールアドレス入力（バリデーション付き）
    • Password: パスワード入力（マスク表示）
    • Number: 数値入力
    • Tel: 電話番号入力
    • URL: URL入力

    4. デザインのポイント

    ラベルとプレースホルダー

    • ラベル: 入力項目の説明を明確に表示
    • プレースホルダー: 入力例やヒントを示す（ラベルの代わりには使わない）
    • 必須マーク: 必須項目には「*」などのマークを表示

    フィードバック

    • リアルタイムバリデーション: 入力中にエラーを表示
    • 成功状態: 正しい入力時に視覚的フィードバック
    • エラーメッセージ: 具体的で解決可能なエラーメッセージ

    レイアウト

    • 適切な幅: 入力内容に応じた幅を設定
    • 間隔: ラベル、入力欄、エラーメッセージの間隔を適切に

    5. アクセシビリティ

    キーボード操作

    • Tabキー: フォーカス可能にする
    • Enter: フォーム送信（フォーム内の場合）

    ARIA属性

    • aria-label: ラベルがない場合に説明を追加
    • aria-describedby: ヘルプテキストやエラーメッセージとの関連付け
    • aria-invalid: バリデーションエラー時
    • aria-required: 必須項目であることを示す

    その他

    • 十分なコントラスト: テキストと背景のコントラスト比を確保
    • フォントサイズ: 小さすぎないサイズ（最低16px）

    6. 誤用・アンチパターン

    よくある間違い

    1. プレースホルダーをラベルの代わりに使用

      • プレースホルダーは入力時に消えるため、ラベルは別途必要

    2. エラーメッセージが抽象的

      • 「無効です」ではなく「正しいメールアドレス形式で入力してください」

    3. 入力制限の説明不足

      • 文字数制限や形式の説明を事前に示す

    4. モバイルでの入力タイプ指定不足

      • type="email"やtype="tel"で適切なキーボードを表示

    7. 関連するUIパターン

    • Form: フォーム内の入力要素として使用
    • Search: 検索バーとして使用
    • Validation: バリデーション機能と組み合わせ
    • Autocomplete: 自動補完機能と組み合わせ
