    Text Link

    本文中やコンテンツ内に配置されるテキストリンク。ナビゲーションリンクとは異なる用途で使用される。

    Text Link

    1. 何のためのコンポーネントか（役割）

    Text Linkは、本文中やコンテンツ内に自然に配置されるテキストリンクです。主な役割は：

    • 関連情報への導線: 記事内の関連ページや参考資料へのリンク
    • 文脈的なナビゲーション: コンテンツの流れを保ちながら別の情報へ導く
    • 引用・参照: 出典や参考資料へのリンク

    Text Linkは、コンテンツの一部として自然に機能し、ユーザーの探索を支援します。

    2. よく使う場面

    • ブログ記事: 関連記事や参考資料へのリンク
    • ドキュメント: 関連セクションや外部リソースへのリンク
    • ニュース記事: 関連記事やソースへのリンク
    • 商品説明: 関連商品や詳細情報へのリンク

    3. 構造と状態

    基本構造

    [通常のテキスト] [リンクテキスト] [通常のテキスト]

    主要な状態

    • Default: 通常のリンク状態（下線付き、色付き）
    • Hover: マウスオーバー時の状態
    • Visited: 訪問済みリンク
    • Focus: キーボードフォーカス時

    4. デザインのポイント

    視覚的識別

    • 下線: 本文中のリンクには下線を使用（特に重要）
    • 色: 背景色と十分なコントラストを確保
    • ホバー効果: マウスオーバー時に色を変える、下線を太くする

    ラベル設計

    • 説明的: リンク先の内容が分かるテキストを使用
    • 自然な流れ: 周囲のテキストと自然に統合される

    5. アクセシビリティ

    • キーボード操作: Tabキーでフォーカス可能
    • 十分なコントラスト: WCAG AA基準（4.5:1以上）を満たす
    • フォーカス表示: 明確な視覚的インジケーター

    6. 誤用・アンチパターン

    • 「ここをクリック」: リンクテキスト自体が説明的であるべき
    • 色だけに依存: 下線やアイコンも使用
    • 長すぎるリンク: 読みやすさを損なう

    7. 関連するUIパターン

    • Article: 記事内のリンク
    • Documentation: ドキュメント内の参照リンク
