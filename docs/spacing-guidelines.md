# スペーシングガイドライン

## 基本方針

### remを使うべき箇所
- **フォントサイズ**: 常にremを使用（ユーザーのフォントサイズ設定に応じてスケール）
- **テキスト周りのスペーシング**: margin, padding（テキストの読みやすさに関わるため）
- **アクセシビリティを重視する要素**: ボタン、リンク、フォーム要素など

### pxを使うべき箇所
- **固定レイアウト**: border-width, border-radius
- **デザインシステムの最小単位**: 1px, 2px, 3pxなどの細かい調整
- **ピクセルパーフェクトが必要な要素**: アイコン、ロゴなど

## スペーシングスケール（8pxベース）

| px値 | rem値 | 用途例 |
|------|-------|--------|
| 4px  | 0.25rem | 最小スペース |
| 8px  | 0.5rem  | 小さなスペース |
| 12px | 0.75rem | 中程度のスペース |
| 16px | 1rem    | 標準スペース |
| 24px | 1.5rem   | 大きなスペース |
| 32px | 2rem     | セクション間のスペース |
| 48px | 3rem     | 大きなセクション間のスペース |
| 64px | 4rem     | ページセクション間のスペース |

## 変換例

### 現在のコード
```css
padding: 32px;
gap: 16px;
margin: 0 0 1.75rem;
```

### 統一後（rem使用）
```css
padding: 2rem;        /* 32px → 2rem */
gap: 1rem;            /* 16px → 1rem */
margin: 0 0 1.75rem;  /* 既にremなのでそのまま */
```

## 実装方針

1. **段階的な移行**: 新規コードからremに統一し、既存コードは段階的に移行
2. **CSS変数での管理**: 将来的にCSS変数でスペーシングを管理することを検討
3. **一貫性の維持**: 同じ用途のスペーシングは同じ値を使用
